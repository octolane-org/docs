All API requests require authentication using an API key. This key should be included in the `x-api-key` header of your requests.

<Info>
  You can get your API key from the [Octolane AI Dashboard](https://app.octolane.com/dashboard/settings)
</Info>

### Authentication Header

Include your API key in the `x-api-key` header with all API requests:

```bash
x-api-key: your_api_key_here
```

### Example Request with Authentication

<CodeGroup>
```bash cURL
curl --request GET \
  --url 'https://enrich.octolane.com/v1/ip-to-company?ip=8.8.8.8' \
  --header 'x-api-key: your_api_key_here'
```

```typescript TypeScript
const headers = {
  'x-api-key': 'your_api_key_here',
  'Content-Type': 'application/json'
};

const response = await fetch('https://enrich.octolane.com/v1/ip-to-company?ip=8.8.8.8', {
  method: 'GET',
  headers: headers
});
```

```python Python
import requests

headers = {
    'x-api-key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://enrich.octolane.com/v1/ip-to-company?ip=8.8.8.8',
    headers=headers
)
```
</CodeGroup>

### Security Best Practices

<Warning>
  Never expose your API key in client-side code or public repositories. Always store it securely in environment variables or secure configuration management systems.
</Warning>

<AccordionGroup>
  <Accordion title="Secure Storage" icon="lock">
    - Store API keys in environment variables
    - Use secure secret management services in production
    - Never commit API keys to version control
  </Accordion>
  
  <Accordion title="Key Rotation" icon="rotate">
    - Regularly rotate your API keys
    - Immediately rotate keys if they're exposed
    - Use different keys for development and production
  </Accordion>
  
  <Accordion title="Access Control" icon="shield">
    - Restrict API key access to necessary team members
    - Use separate keys for different applications or services
    - Monitor API key usage for suspicious activity
  </Accordion>
</AccordionGroup>
